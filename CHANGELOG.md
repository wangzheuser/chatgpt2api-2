## Unreleased

## 2.7.0-rc.1 - 2026-07-10

+ [说明] 这是 `v2.7.0` 的 RC 测试候选版，主要用于提前验证新功能和稳定性；生产环境建议继续使用稳定版，确认无明显问题后再发布正式版。
+ [新增] 对话画图新增本地提示词库工作流，包含默认提示词资产、提示词来源设置、前端选择器和后端管理接口，便于后续继续扩展提示词源、画图参数和超分等能力。
+ [新增] 账号管理新增 OAuth 登录导入入口，补回原版 auth 登录类账号导入路径，减少只能手动填写 token 的场景。
+ [新增] 注册账号新增 DoneMail 邮箱来源，支持通过 DoneMail Admin Key 读取指定收件地址的验证码邮件。
+ [优化] 对话画图交互继续整理：参考图预览、图生图、局部重绘、对比弹窗、图片任务展示和输入区工具入口统一收口到 Studio 组件体系，减少自定义零散样式。
+ [优化] Studio Markdown 和代码块渲染拆分为独立运行时与渲染器，补齐代码格式化、高亮和布局约束，降低长回复、代码块和历史消息渲染对主页面的影响。
+ [优化] 前端大页面按职责拆分运行时：账号、设置、图片管理、日志、注册机、代理、监控和 Studio 页面拆出查询、选择、导入导出、批量操作、详情渲染等模块，降低单文件复杂度并减少页面切换卡顿。
+ [优化] 新增通用前端运行时能力，包括页面查询状态、窗口化列表、空闲任务、统一错误文案、互斥浮层菜单和状态标签色板，为后续继续统一组件样式提供基础。
+ [优化] 状态标签、额度标签、弹窗关闭按钮、浮层菜单和实时监控展示样式统一，减少“同类状态不同视觉”的问题。
+ [优化] 概览指标写入增加缓冲处理，并补充后端流程、并发架构、前端性能、数据存储、图片链路和开发约束文档，方便后续继续做高并发和高成功率优化时先对齐边界。
+ [优化] 系统设置保存后增加账号清理确认流程，避免开启自动移除类开关后用户无感触发危险操作。
+ [优化] Sentinel SDK 地址更新到 `20260219f9f6`，保持注册链路依赖的前端采集脚本版本更接近当前官方页面。
+ [修复] 图片生成失败后会触发一次远程账号信息刷新，用于尽快确认坏号、额度耗尽或状态异常；该流程只刷新状态，不直接删除账号，删除仍受既有自动移除开关控制。
+ [修复] 注册机远程账号确认逻辑收口，减少远程导入/刷新后状态误判。
+ [修复] 图片流式响应不再把最终图片同时作为 `partial_image` 和 `completed` 各发送一次，避免客户端看到两张一样的最终图。

## 2.6.2 - 2026-07-08

+ [修复] OpenAI 注册流程 `create_account` 阶段的 `registration_disallowed` 错误，补齐 Turnstile SO Token 生成与双 Sentinel header 发送，对齐官方 SDK 5000ms 采集行为，预计成功率从 0% 提升至 50% 左右。
+ [优化] Turnstile Token 生成器完整重写，实现支持递归队列和所有 VM 操作码的解释器，提升 Sentinel Token 生成的稳定性和兼容性。
+ [优化] Sentinel Token 生成新增 `build_sentinel_with_so_token()` 函数，同时返回独立的 SO Token 以支持需要单独 header 的场景。

+ [优化] 对话画图图片放大弹窗复用图片管理弹窗样式，预览、复制和下载交互保持一致。
+ [优化] 注册机浏览器指纹改为多套 profile 轮换，降低固定指纹被复用导致的注册失败概率。
+ [修复] 图片链路文本回复识别进一步收口，基于结构化消息、文件产物和任务状态判断真正“文本回复但无图”的场景；正常排队/处理中提示继续轮询，避免误判为错误或一直等到超时。

## 2.3.4 - 2026-06-26

+ [新增] 文本链路支持 `reasoning_effort`、`thinking_effort` 和 `reasoning.effort` 透传，`/v1/chat/completions` 与 `/v1/responses` 都会传到 ChatGPT 会话 payload，兼容需要推理强度参数的上游模型。

+ [新增] 新增日志保留天数配置 `log_retention_days`，服务启动和后台定时任务会自动清理过期 `data/logs.jsonl`，减少长期运行后的日志体积。

+ [优化] 设置页说明文案改为问号悬浮提示，账号刷新、图片清理、轮询超时、上游流超时、单账号图片并发、Cloudflare 清障、代理运行时等字段说明更清爽。
